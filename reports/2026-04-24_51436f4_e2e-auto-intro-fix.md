## Pre-Existing Issue: Skill Discovery Returns 0

Console logs show both installs succeed but discovery finds nothing:
```
[daytona] createHermesSandbox npx skills add succeeded: 12714ms
[daytona] createHermesSandbox symlinked to hermes skills dir: 12981ms
[daytona] createHermesSandbox discovered 0 skills on disk: 16234ms

[daytona] installSkill npx skills add succeeded: 4708ms
[daytona] installSkill symlinked to hermes skills dir: 5009ms
[daytona] installSkill discovered 0 skills on disk: 10636ms
```

**Root cause hypothesis**: `npx skills add` may install to a different subdirectory structure than expected, or the symlink target path doesn't match. The discovery command `for d in /root/.hermes/skills/*/; do [ -f "$d/SKILL.md" ] && basename "$d"; done` finds no SKILL.md files despite the symlink step reporting success.

**Impact**: Hermes falls back to its default skill (`hermes-agent`) instead of loading the requested skill. This is NOT caused by PR#32.
